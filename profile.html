              Verification Model
              ▼
            

            

              

                
Layer 1 — ECDSA Signatures

                  Each award is signed by the repo owner's key. Forgery requires their private key.

                
Layer 2 — Double-entry

                  AppView (Bobbin) cross-checks both the hunter's PDS and the owner's PDS before indexing.

                
Layer 3 — Authority Weight

                  Points scaled by repo credibility (stars, age, contributors). A sock-puppet repo awards near-zero points.

                
Layer 4 — Git Auditability

                  Every award links to a real commit hash. Fake PRs leave a permanent public trail.
